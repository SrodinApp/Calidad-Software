La importancia de la calidad de software

Cuando se habla de la calidad de un producto, rápidamente pensamos en la fabricación de este, determinando en la durabilidad y el cumplimiento de la función por la cual fue fabricado dicho producto. Pero cuando trasladamos este concepto al desarrollo de software, el mismo termino debe verse ampliado ya que por las características intangibles de un producto de software la primera definición no se le puede aplicar completamente. Por otro lado, en un producto de software intervienen otros aspectos que determinan su calidad., Tales aspectos a mencionar son; seguridad, flexibilidad, complejidad, adaptabilidad, eficiencia, facilidad para usar., Entre otros aspectos. Mencionado estos conceptos hay que hablar sobre los paradigmas de programación que surgieron como consecuencia de la calidad deficiente del software, pero antes que nada ¿Qué son paradigmas? Y que función cumplen dentro del desarrollo.

Paradigmas de programación

Paradigma de programación según la definición de la RAE, un paradigma es “Ejemplo o ejemplar” (2021). Bajo esta premisa podemos decir que un paradigma de programación es un ejemplo para el desarrollo de software entonces la función de un paradigma es estandarizar un método para crear esa similitud de desarrollo entre programadores, de esta forma facilitar la tarea no solo del desarrollo y construcción si no de la flexibilidad de los proyectos de software. Entendiendo que el desarrollo de software es una tarea meramente creativa y se ha evidenciado que los programadores para llegar a una solución tomaban distintos caminos entre uno y otro. Como se menciona en el SWEBOK (2014). “Esta diversidad de programación causo mucha dificultad en la construcción y mantenimiento de software grande y complejo” (p. 228). A lo largo del tiempo han surgido distintos paradigmas los cuales son:

  • programación no estructurada
  • programación estructurada / procedimental o imperativa
  • programación orientada a objetos
  • programación orientada a aspectos
  • programación funcional

Programación no estructurada

Este tipo de programación es conocida por producir código difícilmente legible ya que como se define en el SWEBOK “Los programas escritos de esta manera no exhiben una estructura particular, de ahí su nombre”. Como ejemplos de lenguajes de programación que tienen este paradigma son:

  • Basic
  • Fortran

Programación estructurada, procedimental o imperativa

Este paradigma de programación contrasta con la programación no estructurada ya que como se define en el SWEBOK la principal característica de este tipo de programación es “el uso de estructuras de control bien definidas, que incluyen procedimientos y cada procedimiento realiza una tarea especifica” (p. 229). Este tipo de programación surgió debido a la falta de claridad, calidad y de tiempo de desarrollo de otros tipos de programación, con este tipo de programación se siguen reglas generales y algunos protocolos establecidos que ayuda a que el código sea más fácilmente legible. Como ejemplos de este tipo de programación esta:

  • C
  • Java
  • Python

Programación orientada a objetos

Es bien conocida por sus siglas en ingles O.O.P. (Object, Oriented, Programming). En este paradigma de programación se modelan los elementos a partir de clases, luego se crean los objetos como instancias de la clase. Como característica principal de este tipo de programación es la creación de objetos que representa una entidad abstracta que interactúan entre sí para cumplir en conjunto las funciones objeto. Como ejemplo:

  • JavaScrip
  • Java
  • Python
  • C++

Programación orientada a aspectos

La programación orientada a aspectos o en sus siglas AOP. Es un tipo de programación que permite separar componentes y aspectos y estos entre sí y viceversa o como se menciona en el SWEBOK la principal característica “Es resolver el enredo y la dispersión de los objetos asociados con O.O.P. en los que las interacciones entre objetos se vuelven muy complejas” (p. 229). Como ejemplos de lenguajes de programación hay:

  • Sprint Python
  • Aspect C++
  • Aspect J

Programación funcional

La programación funcional o por sus siglas F.P. es un tipo de programación en la cual se emplean mayoritariamente funciones evitando los datos mutables y también se evita compartir estados entre las funciones, proporcionando una transparencia referencial. Como se menciona en el SWEBOK “En la programación funcional todos los cálculos se tratan como la evaluación de funciones matemáticas” (p. 229). Ejemplos de lenguajes de en que se puede emplear este tipo de programación son:

  • Scala
  • Lisp
  • Clojure

El papel tiene la calidad en el desarrollo de software

Independientemente del paradigma de programación a emplear, el aspecto debe estar presente no solo en el memento de escribir el código si no en todo el proyecto de software. Para lograr los objetos de calidad de un producto Pressman (2010) expone unos aspectos bastante interesantes para alcanzar los objetivos de calidad propuestos, en su libro expone los siguientes aspectos:

Primero debe crearse una cantidad de actividades y procedimientos que le ayuden al ingeniero de software a generar un producto de calidad, en segundo lugar, se debe implementar el control de calidad y las actividades relacionadas a este aspecto para asegurar la calidad de dicho producto y lo último es implementar métricas para desarrollar estrategias con el objetivo del mejoramiento del desarrollo de software y de esta manera alcanzar el objetivo de calidad propuesto. (p. 368)
